# Grabia Auto-Tag Key
# =====================
# Maps filename tokens to structured tags. Tokens are extracted from
# parentheses (...) and square brackets [...] in filenames.
#
# This file has two sections:
#   1. STATIC MAPPINGS  — exact text matches (case-insensitive)
#   2. REGEX PATTERNS   — regular expression rules for dynamic matching
#
# ─── Static Mapping Format ─────────────────────────────────────────────
#   TOKEN > tag1, tag2, ...
#
#   TOKEN is matched case-insensitively against each extracted token.
#   Tags use parent:child format (e.g. region:japan) or standalone (e.g. demo).
#   Multiple tags can be comma-separated.
#
# ─── Regex Pattern Format ──────────────────────────────────────────────
#   /REGEX/FLAGS > tag_template
#
#   Lines starting with / are treated as regex rules.
#   FLAGS can include: i (case-insensitive)
#   Tag templates can reference capture groups:
#     {0}  = entire match
#     {1}  = first capture group, {2} = second, etc.
#   Modifiers on groups:
#     {1}        = raw capture group
#     {1:lower}  = lowercased
#   Regex rules are tried in order after all static lookups fail.
#
# ─── Notes ─────────────────────────────────────────────────────────────
#   - Blank lines and lines starting with # are ignored
#   - Tokens are extracted from both (...) and [...] in filenames
#   - Comma-separated values inside parens are split:
#       (USA, Europe) → tokens: "USA", "Europe"
#   - Unknown tokens become unknown:token_here

# ===========================================================================
#  REGEX PATTERNS
# ===========================================================================
# These are tried IN ORDER when a token doesn't match any static mapping.
# Use /regex/flags > tag_template format.
# Capture groups: {1}, {2}, etc. Modifiers: {1:lower}

# ─── Revisions ────────────────────────────────────────────────────────────
# (Rev A), (Rev B), (Rev 1), (Rev 2.1), etc.
/^Rev\s+(.+)$/i                            > alt:rev_{1:lower}

# ─── Versions ─────────────────────────────────────────────────────────────
# (v1.0), (v2.1), (V1.0.1), (v21) — with v/V prefix
/^[vV](\d[\d.]*)$/                          > version:{1}

# (v1.4 Beta), (v2.0 hotfix 3), (v1_hotfix) — version with suffix text
/^[vV](\d[\d.]*)[_ ].+$/                    > version:{1}

# Bare version numbers without v prefix: (1.1.6), (2.0.1)
/^(\d+\.\d[\d.]*)$/                         > version:{1}

# ─── Alternates ───────────────────────────────────────────────────────────
# (Alt 1), (Alt 2), etc.
/^Alt\s+(\d+)$/i                            > alt:{1}

# ─── Dates ────────────────────────────────────────────────────────────────
# (1992-09-xx), (2003-01-15), (1996-03-XX)
/^(\d{4})-(\d{2})-([0-9xX]{2})$/           > date:{0:lower}

# (19920901), (20030115) — compact YYYYMMDD
/^(\d{4})(\d{2})(\d{2})$/                   > date:{1}-{2}-{3}

# ─── Disc / Side / Track ──────────────────────────────────────────────────
# (Disc 1), (Disk 2), etc. — catches any disc number
/^Dis[ck]\s+(\d+)$/i                        > disc:{1}

# (Side A), (Side B), etc.
/^Side\s+([A-Za-z])$/i                      > side:{1:lower}

# (Track 1), (Track 2), etc.
/^Track\s+(\d+)$/i                          > track:{1}

# ─── Translations ─────────────────────────────────────────────────────────
# [T+Eng], [T+Fre], etc. — catch-all for unlisted translation codes
/^T\+(.+)$/i                                > translation:{1:lower}

# ─── Bracket dump codes with numbers ──────────────────────────────────────
# [b1], [b2], [h1], [h2], [a1], etc.
/^([bhoa])(\d+)$/i                          > {1:lower}_dump:{2}
